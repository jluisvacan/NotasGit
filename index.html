Git

Core concepts

working directory

Es la carpeta del proyecto donde el usuario edita directamente los archivos

  • Contiene los archivos locales
  • Git detecta los cambios (modificaciones, archivos nuevos o eliminados)
  • No esta en el historial hasta que se realiza un add o un commit

staging area

Es una lista temporal donde Git ubica los cambios que se quieren incluir en el proximo commit

  • No se guardan los cambios, solo se preparan
  • Permite hacer commits selectivos
  • Se usa el comando git add [nombre_de_archivo]

repository

Es donde Git almacena el historial de versiones del proyecto

  • Se crea con git init (de manera local) o git clone (desde remoto)
  • Guarda todos los commits
  • Se puede conectar a un repositorio remoto

Essencial Commands

git init

Inicializa un nuevo repositorio Git en la carpeta del proyecto.

git status

Muestra el estado actual del repositorio.

git add archivo.txt

Prepara los archivos para ser parte del proximo commit, los agrega al staging area.

git commit -m "message"

Guarda los cambios preparados con un mensaje descriptivo.

git diff

Muestra las diferencias entre los cambios en el working directory comparados con lo del ultimo commit.

git diff --cached

Muestra las diferencias entre los archivos del staging area y el ultimo commit.

git restore

Deshace los cambios en archivos individuales.

git reset

Mueve la rama actualr (HEAD) a otro commit.

git revert

Crea un nuevo commit que revierte los cambios de un commit anterior sin eliminarlo del historial.

git branch

Lista las ramas locales

git switch -c nameBranch

Crea y cambia a una nueva rama con el nombre nameBranch.

git branch -D nameBranch

Elimina una rama sin importar que se haya o no hecho merge previo.

Merging branches

git merge nameBranch

Combina los cambios de otra rama en la rama actual

git merge --continue

Despues de resolver un conflicto de merge, permite continuar con el proceso.

git merge --abort

Cancela una fusion con conflictos antes de completarla.

git rebase nameBranch

Replica los commits de la rama actual sobre la rama indicada.

git rebase --continue

Despues de resolvcer un conflicto de reabase, permite continuar con el proceso.

git rebase --abort

Cancela el proceso de rebase con conflictos sin resolver.

git rebase -i HEAD~n

Rebase interactivo, permite editar combinar, reordenar y rebase, menteniendo un historial lineal.

Remote repository

git remote add origin repositoryURL

Agrega un repositorio remoto.

git push origin main

Envia los cambios al repositorio remoto.

git pull origin main

Descarga los cambios en el remoto y los fusiona con la rama indicada.

git fetch origin nameBranch

Descarga los cambios a la rama remota indicada sin realizar el merge con la rama local.

git remote remove origin

Elimina el repositorio remoto.
